# 电商经营首诊报告商业安全审稿标准 V1.0

## 1. 报告定位

首诊报告用于陌生电商商家的首次触达，不是完整咨询报告。目标是：

- 让客户感觉我们看懂了经营问题；
- 不乱下结论，不伪造行业标准；
- 提出低风险验证动作；
- 引导客户愿意继续提供后台数据；
- 自然承接 4 周经营追踪、7+7 周报、月度复盘和后续自动化运营模块。

## 2. 版本评级

- 内部验收稿：方向可用，但存在明显强判断、动作风险或客户阅读障碍。
- 试点客户版：商业安全边界基本成立，可小范围发真实商家测试反馈。
- 正式客户版：表达安全、客户可读、动作低风险、后续承接自然，适合稳定交付。

## 3. 商业安全审稿标准

1. 无活动前后对比、历史同期数据、渠道拆分或归因数据时，不得写“证明、已经带动、跑通、导致、造成”等强因果判断。应改为“在……背景下观察到”“具备观察价值”“需进一步验证贡献”。
2. 无目标值、历史均值、行业均值或同层级对标时，不得写“未达到预期、低于预期、成交缺口、转化率偏低、流量规模达标”等预期判断。应改为“当前样本显示”“仍有拆解空间”“需进一步对标判断”。
3. 无历史均值、行业均值、平台同层级数据或连续多周数据时，不得写“稳定、无异常、健康、合理、优秀、达标、偏低、偏高”等状态判断。应改为“本次样本暂未显示明显风险”“当前提交数据中未见明显风险信号”“仍需结合历史数据判断”。
4. 无上周、上月、活动前后、历史均值或连续多周数据时，不得写“上涨、下滑、改善、恶化、变好、变差、同步放大、没有跟上”等趋势判断。只能写当前数值和待验证方向。
5. 若第 07 条数据边界声明不输出“达标/合理/优秀/偏低/砍预算/加预算”等强判断，正文前文也不得出现同类表达。
6. 无阈值来源时，不得写 60%、70%、80%、30% 等固定判断线。若必须保留，应注明“本次试点观察线，仅用于初步排查，不代表行业标准”。
7. 无 ROI 口径、毛利率、渠道拆分、广告直接成交和间接成交数据时，不得直接建议加预算、砍预算、停投或放量。
8. 涉及页面替换、预算调整、投放定向、价格机制、库存调整、停推新品或全店改版时，只能写成“小范围测试/具备条件时验证/验证后再决定”，不得直接建议全量执行。
9. 每个核心判断必须区分：确定事实、初步观察、高概率假设、待验证数据、低风险验证动作。
10. 对外客户版不得暴露内部工具链、Prompt、模型、GitHub Actions、Notion、豆包、火山方舟等内部实现。

## 4. 高风险词与安全替代表达

| 高风险表达 | 安全替代表达 |
|---|---|
| 证明活动有效 | 在活动背景下观察到变化，仍需验证贡献 |
| 已经带动访客上涨 | 本周在该活动背景下，访客规模达到 X |
| 流量规模达标 | 访客规模已具备分析价值 |
| 未达到预期 | 当前样本显示仍有拆解空间 |
| 转化率偏低 | 是否低于合理水平需结合历史或行业数据判断 |
| 客单价稳定 | 本周客单价为 X 元，稳定性需结合历史数据判断 |
| 无售后异常 | 本次样本暂未显示明显售后风险 |
| ROI 偏低 | ROI 口径未统一，暂无法判断效率 |
| 建议加预算/砍预算 | 先拆分渠道表现，再判断是否具备小范围调整条件 |
| 直接改详情页 | 先形成优化方案，具备条件时小范围验证 |

## 5. 固定审稿输出格式

每次审稿应输出：

1. 当前版本评级：内部验收稿 / 试点客户版 / 正式客户版；
2. 总体判断；
3. 主要优点；
4. 不能直接发客户的风险句；
5. 每个风险句的替换建议；
6. 自动化模板或 GitHub 规则需要补充的地方；
7. 最终是否建议进入真实商家测试。

## 6. 固定审稿 Prompt

请按《电商经营首诊报告商业安全审稿标准 V1.0》检查这份首诊报告。审稿目标：这份报告用于陌生电商商家的首次触达，不是完整咨询报告。目标是让客户觉得我们看懂了经营问题，但不过度下结论；能提出低风险验证动作，并自然引导进入 4 周经营追踪、7+7 周报、月度复盘和后续自动化运营模块。

请重点检查：

1. 是否存在无依据的因果判断，如“证明、已经带动、跑通、导致、造成”等。
2. 是否存在无目标值支撑的预期判断，如“未达到预期、低于预期、成交缺口、转化率偏低、流量规模达标”等。
3. 是否存在无历史/行业数据支撑的状态判断，如“稳定、无异常、健康、合理、优秀、达标、偏低、偏高”等。
4. 是否存在无对比数据支撑的趋势判断，如“上涨、下滑、改善、恶化、同步放大、没有跟上”等。
5. 是否存在正文与第 07 条数据边界声明冲突。
6. 是否存在未说明来源的固定阈值，如 60%、70%、80%、30% 等。
7. 是否存在高风险动作，如加预算、砍预算、停投、降价、全店改页面、停推新品、调整库存等。
8. 是否适合对外客户阅读：语言专业、克制、清楚，避免过度销售，避免 AI 模板感，避免内部产品语言。
